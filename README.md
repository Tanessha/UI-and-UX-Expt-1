# UI-and-UX-Expt-1

## Aim:
The content is based on handwritten notes that were digitized into PDF and focuses on definitions, principles, processes, and tools used in modern UI/UX design.

## Contents
Fundamentals of UI and UX, including clear definitions and differences between UI (visual interface) and UX (overall experience).

Core principles of UX: user‑centred design, usability, accessibility, consistency, feedback, efficiency, and learnability.

Core principles of UI: clarity, visual hierarchy, consistency, readability, affordance, and responsiveness.

UX design process: idea → research → problem definition → user flows & information architecture → wireframes → prototypes → usability testing → iteration.

UI design process: brand guidelines → layout & grid → typography & colour → components → interactions → design system.

Relationship between UI and UX
The notes explain how UI and UX work as two layers of a single design system:

UX defines structure, logic, and flow.

UI defines the visual representation and interaction details.

A combined flow is documented, showing steps from understanding users and defining goals to creating wireframes, visual design, interactive prototypes, and running usability tests with visual refinements.

## Case Studies
The experiment includes short case studies that apply UI and UX thinking:

### E‑commerce checkout:

UX focuses on a short, linear, predictable checkout, reducing surprise charges and abandonment.

UI focuses on clear labels, large form fields, primary buttons like “Place Order”, and visible error messages.

### Mobile banking app (send money flow):

UX looks at what users need after login and designs a simple, secure flow for sending money with minimal steps.

UI designs a clean dashboard, emphasises key actions, and uses colour and typography to convey trust.

## Design Tools Covered
The document briefly describes several UI/UX tools and their strengths in a typical workflow:

Figma – UI design, prototyping, collaboration, and design systems.

Adobe XD – visual UI design with interactive prototypes and animations.

Penpot – open‑source UI design tool, browser‑based with web‑standard compatibility.

Sketch – UI design tool for macOS with symbols and plugins.

Framer – highly interactive web prototypes with production‑like feel.

A typical tool flow goes from research and wireframes (e.g., Balsamiq/Whimsical) to UI design and prototyping (Figma/XD/Sketch/Penpot/Framer) and finally to user testing and developer handoff.

## Result: 
Thus the documented flows and principles can be referenced when designing or reviewing UI/UX projects.
